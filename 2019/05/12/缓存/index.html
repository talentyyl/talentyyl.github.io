缓存

发表于 2019-05-12
  • 为什么使用缓存

提高系统运行效率，减轻数据库压力

  1. ASP.NET 内置缓存

非Web环境下可以使用 HttpRuntime.Cache

  1. NoSQL 数据库

ASP.NET 内置的Cache是“内存缓存”，是放在Web服务器中的，在多台 Web 服务器组成的集群中，无法实现共享，而且如果数据量大会占用 Web 服务器内存。

现在流行的实现“进程外Cache”的服务器有 Redis 和 Memcached，它们都是 NoSQL （Not Only SQL）数据库，不是替代 SQL 数据库，而是为了弥补 SQL 数据库的不足，NoSQL 能做的 SQL 数据库也能做，不过 NoSQL 数据库的效率更高。Redis 可以把数据持久化到硬盘中，Memcached 是放在内存中，重启后就消失，如果仅仅作为 KV 键值对缓存使用，用 Memcached 就可以，如果还要使用 list、set、hash 等或数据不能丢失就要使用 Redis 了。

NOSQL 数据库也是独立运行的服务器，我们的程序通过 SDK （类似于 ADO.NET 之于 SQLServer）连接服务器

  • Memcached
    • Memcached 服务器运行，Memcached 推荐安装在Linux服务器上，Windows 下仅供测试
