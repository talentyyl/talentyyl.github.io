委托事件

发表于 2019-05-10
  • 委托
    • 委托可以理解为指向一个方法的数据类型，类似于 C/C++ 的函数指针，使用委托可增强灵活性和减少重复代码（以冒泡排序的升序和降序为例），委托的声明使用 delegate 关键字，委托实际上是特殊的类，.NET 中所有委托类型总是派生自 System.MulticastDelegate，后者又从 System.Delegate 派生，为了减少自定义委托的必要，C#3.0 提供了泛型委托 Action 和 Func ，委托的实例化可使用 C#2.0 提供的匿名方法和C#3.0 提供 Lambda 表达式 ，新代码优先使用 Lambda 表达式；Lambda表达式和匿名方法并不是CLR的固有构造，相反，当编译器遇到匿名方法时，会把它转化成特殊的隐藏类、字段和方法。

委托实现发布/订阅

  • 事件

    • 委托本身又是 pub/sub 模式的基本单位，使用委托实现发布订阅，事件的发布和订阅都不能得到充分的控制。因此 C# 引入了事件，封装订阅和发布。事件内部就是一个private的委托和 add、remove 方法。事件限制外部类只能通过“+= ”操作符发布者添加订阅方法，并用”-=”取消订阅。
